## Codebase Patterns
- Run tests with `.venv/bin/pytest` (not `python -m pytest`) ŌĆö system Python on macOS lacks pytest
- Always install both requirements.txt AND requirements-dev.txt: `validators` is used in src but only in dev requirements
- src/ modules import each other directly (no package prefix) ŌĆö add `sys.path.insert(0, "src/")` in test files
- conftest.py fixtures: `tmp_config` returns str path; `tmp_podcast_dir` returns pathlib.Path
- sample_feed.xml has 3 episodes; episode 2 title contains `/`, `:`, `?` for sanitization testing
- `{yyyy-mm-dd}` as a single token is NOT replaced by utils.py; real date pattern is `{yyyy}-{mmmm}-{dddd}` (returns unpadded ints: `2024-6-15`)
- `define_string_tokens` date fields return raw integers (tm_year, tm_mon, tm_mday) ŌĆö no zero-padding
- RSS `<description>` in channel maps to `feed.subtitle` in feedparser, NOT `feed.summary` ŌĆö use `<itunes:summary>` for the summary field
- `simplify_metadata()` requires `feed.summary` ŌĆö sample_feed.xml must have `<itunes:summary>` or tests will fail
- For mocking objects with `'key' in obj` checks (feedparser FeedParserDict interface), use a dict subclass with `__getattr__`, not `types.SimpleNamespace`
- `responses.add()` allows unlimited calls to same URL by default ŌĆö no need to re-add for multiple calls in one test
