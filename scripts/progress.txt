## Codebase Patterns
- Run tests with `.venv/bin/pytest` (not `python -m pytest`) — system Python on macOS lacks pytest
- Always install both requirements.txt AND requirements-dev.txt: `validators` is a runtime dep but was only in dev requirements — now fixed in requirements.txt
- For E2E Docker tests: `subscribe.main()` downloads ALL episodes before server starts — use local HTTP server (via `host.docker.internal`) not real podcast feeds
- Use `--add-host host.docker.internal:host-gateway` in `docker run` to let containers reach the test host
- src/ modules import each other directly (no package prefix) — add `sys.path.insert(0, "src/")` in test files
- conftest.py fixtures: `tmp_config` returns str path; `tmp_podcast_dir` returns pathlib.Path
- sample_feed.xml has 3 episodes; episode 2 title contains `/`, `:`, `?` for sanitization testing
- `{yyyy-mm-dd}` as a single token is NOT replaced by utils.py; real date pattern is `{yyyy}-{mmmm}-{dddd}` (returns unpadded ints: `2024-6-15`)
- `define_string_tokens` date fields return raw integers (tm_year, tm_mon, tm_mday) — no zero-padding
- RSS `<description>` in channel maps to `feed.subtitle` in feedparser, NOT `feed.summary` — use `<itunes:summary>` for the summary field
- `simplify_metadata()` requires `feed.summary` — sample_feed.xml must have `<itunes:summary>` or tests will fail
- For mocking objects with `'key' in obj` checks (feedparser FeedParserDict interface), use a dict subclass with `__getattr__`, not `types.SimpleNamespace`
- `responses.add()` allows unlimited calls to same URL by default — no need to re-add for multiple calls in one test
- `.venv/bin/python` links to python3.13 but packages are installed under python3.12 — use `.venv/bin/python3.12` or `.venv/bin/pytest` for manual testing
- `utils.define_string_tokens` uses `entry.get("episode", None)` but feedparser stores `<itunes:episode>` as `itunes_episode` — episode token is always empty for real feedparser entries
